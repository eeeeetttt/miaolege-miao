# 喵了个喵 - 星球跟单平台 业务逻辑文档

## 一、平台概述

### 1.1 平台定位
喵了个喵是一个基于交易信号跟单的社区化平台，旨在帮助交易者更好地分享和跟随优质的交易信号。

### 1.2 核心功能模块
1. **用户系统** - 注册、登录、个人资料管理
2. **星球系统** - 创建、管理、加入星球
3. **信号系统** - 信号发布、接收、展示
4. **跟单系统** - 跟单、暂停、恢复
5. **MT账号系统** - MT4/MT5账号绑定
6. **星球币系统** - 充值、消费、余额管理
7. **EA产品系统** - EA产品购买、下载
8. **后台管理** - 用户管理、系统配置、数据统计

## 三、核心业务流程

### 3.1 用户注册与登录

#### 注册流程
```
1. 用户填写邮箱、密码
2. 系统验证邮箱格式、密码强度
3. 检查邮箱是否已注册
4. 密码加密存储（bcrypt）
5. 创建用户记录，初始星球币为0，角色为user
6. 自动登录，跳转到首页
```

#### 登录流程
```
1. 用户填写邮箱、密码
2. 系统验证邮箱是否存在
3. 验证密码是否正确
4. 创建Session（NextAuth）
5. 跳转到首页或之前访问的页面
```

---

### 3.2 MT账号绑定流程

#### 绑定流程
```
1. 用户进入个人中心 -> MT账号绑定
2. 选择平台类型（MT4/MT5）
3. 输入MT账号
4. 系统检查：
   - 用户是否已绑定其他MT账号（每人只能绑定一个）
   - 该MT账号是否已被其他用户绑定
5. 创建绑定记录，isVerified=false
6. 绑定成功，可用于接收跟单信号
```

#### 解绑流程
```
1. 用户点击解绑按钮
2. 确认提示
3. 删除绑定记录
4. 解绑成功后，无法再接收跟单信号
```

**重要规则**：
- 每个用户只能绑定一个MT账号
- MT账号绑定的主要作用是让EA能够识别跟单用户

---

### 3.3 星球创建流程

#### 创建流程
```
1. 用户点击"创建星球"
2. 填写星球信息：
   - 名称（必填）
   - 描述
   - 规则
   - 门票价格（默认0）
   - 邀请码（可选）
   - 最大发布者数量（默认3）
   - 时长选择：7天/1年/3年/永久
3. 系统根据时长计算费用：
   - 7天：免费（0星球币）
   - 1年：1999星球币
   - 3年：2999星球币
   - 永久：4999星球币
4. 检查用户星球币余额是否足够
5. 扣除费用（如果有）
6. 创建星球记录：
   - durationDays = 0 表示永久
   - expireAt = null 表示永久
   - 其他情况计算过期时间
7. 添加创建者为星球owner
8. 创建成功
```

#### 星球过期处理
```
- durationDays = 0 且 expireAt = null：永久有效
- durationDays > 0：根据expireAt判断是否过期
- 过期星球应该被标记为closed状态
```

---

### 3.4 加入星球流程

#### 方式一：购买门票加入
```
1. 用户浏览星球列表
2. 点击"加入星球"
3. 选择"购买门票"
4. 系统检查：
   - 是否已是该星球成员
   - 用户星球币余额是否足够支付门票
5. 扣除门票费用
6. 记录星球收入（planet_earnings）
7. 添加成员记录：
   - role = 'follower'
   - joinMethod = 'purchase'
   - ticketPaid = 门票价格
   - expiryDate = 当前时间 + 星球时长
8. 加入成功
```

#### 方式二：邀请码加入
```
1. 用户点击"加入星球"
2. 选择"邀请码"
3. 输入邀请码
4. 系统验证邀请码是否正确
5. 添加成员记录：
   - role = 'follower'
   - joinMethod = 'invite'
   - ticketPaid = 0
   - expiryDate = 当前时间 + 星球时长
6. 加入成功
```

---

### 3.5 信号发布流程

#### EA发送信号到平台
```
1. EA检测到交易信号（开仓/平仓等）
2. EA调用API：POST /api/signal/receive
3. 请求参数：
   - signal_type: 信号类型
   - sender_account: MT账号
   - symbol: 交易品种
   - order_type: 订单类型
   - volume: 手数
   - price: 价格
   - sl: 止损
   - tp: 止盈
   - deal_profit: 平仓盈亏
   - balance: 账户余额
   - broker: 经纪商
   - planet_id: 星球ID（可选）
   - comment: 备注

4. 系统处理：
   a. 如果提供了planet_id，直接关联
   b. 如果没有planet_id：
      - 通过sender_account查找绑定的用户
      - 查找该用户作为owner/publisher的星球
      - 自动关联到星球

5. 保存信号记录
6. 返回成功响应
```

#### 信号自动关联规则
```
优先级：
1. 显式指定的planet_id
2. 用户作为publisher的星球
3. 用户作为owner且开启ownerAsPublisher的星球
```

---

### 3.6 跟单流程

#### 开始跟单
```
1. 用户进入星球 -> 信号源详情页
2. 查看信号源历史信号和收益统计
3. 点击"开始跟单"
4. 系统检查：
   - 用户是否已登录
   - 用户是否是该星球成员
   - 用户是否已绑定MT账号
5. 显示跟单确认弹窗（星球成员免费跟单）
6. 用户确认
7. 创建跟单记录：
   - planetId: 星球ID
   - userId: 用户ID
   - signalId: 最新的平仓信号ID
   - status: 'active'
   - copyRatio: 跟单比例，默认1.00
8. 跟单成功
```

#### 暂停跟单
```
1. 用户点击"暂停跟单"
2. 更新跟单记录：
   - status: 'paused'
   - pausedAt: 当前时间
3. 暂停成功，EA将不再执行跟单
```

#### 恢复跟单
```
1. 用户点击"恢复跟单"
2. 更新跟单记录：
   - status: 'active'
   - pausedAt: null
3. 恢复成功，EA将继续执行跟单
```

---

### 3.7 EA验证跟单权限流程

#### EA调用验证接口
```
1. EA调用：GET /api/purchase/validate?followAccount=MT账号
2. 系统处理：
   a. 通过MT账号查找绑定的用户
   b. 检查用户是否有活跃的跟单记录
   c. 检查用户是否是星球成员

3. 返回结果：
   - valid: true/false
   - hasPurchase: true/false（是否有跟单权限）
   - status: 'active'/'paused'
   - planetId: 星球ID
   - planetName: 星球名称
   - followId: 跟单记录ID

4. EA根据返回结果决定是否执行跟单
```

#### 验证逻辑
```
用户有跟单权限的条件：
1. MT账号已绑定到平台用户
2. 该用户是某星球的成员
3. 该用户有active状态的跟单记录
```

---

### 3.8 星球币充值流程

```
1. 用户进入个人中心 -> 充值
2. 选择充值金额（10-50000）
3. 选择支付方式
4. 创建充值记录，status='pending'
5. 调用支付系统（当前为模拟，直接完成）
6. 支付成功后：
   - 更新充值记录status='completed'
   - 增加用户星球币余额
7. 充值完成
```

---

### 3.9 EA产品购买流程

```
1. 用户浏览EA产品列表
2. 查看产品详情
3. 点击购买
4. 系统检查：
   - 用户是否已登录
   - 产品是否上架
   - 用户是否已购买过
   - 用户星球币余额是否足够
5. 扣除星球币
6. 创建购买记录
7. 更新产品销量
8. 购买成功，可下载EA
```

## 四、权限与角色系统

### 4.1 用户角色

| 角色 | 说明 | 权限 |
|------|------|------|
| user | 普通用户 | 注册、登录、加入星球、跟单、购买EA、充值 |
| admin | 管理员 | 所有user权限 + 后台管理、用户管理、系统配置 |

### 4.2 星球角色

| 角色 | 说明 | 权限 |
|------|------|------|
| owner | 星主 | 管理星球、审核发布者申请、设置门票价格、邀请码 |
| publisher | 发布者 | 在星球内发布信号 |
| follower | 跟单者 | 查看信号、跟单 |

### 4.3 权限检查点

```
创建星球：
  - 需要登录
  - 需要足够的星球币支付费用

加入星球：
  - 需要登录
  - 非该星球成员
  - 门票支付或邀请码验证

跟单：
  - 需要登录
  - 需要是星球成员
  - 需要绑定MT账号

管理星球：
  - 需要是星球owner

审核发布者：
  - 需要是星球owner

发布信号：
  - 需要是星球owner或publisher
```

## 七、核心业务规则

### 7.1 星球创建规则

```
价格规则：
- 7天：免费（0星球币）
- 1年：1999星球币
- 3年：2999星球币
- 永久：4999星球币

过期规则：
- durationDays = 0 且 expireAt = null：永久有效
- durationDays > 0：expireAt = 创建时间 + durationDays天
- 过期后星球状态变为closed
```

### 7.2 门票规则

```
- 星主可设置门票价格（星球币）
- 用户通过购买门票加入星球
- 门票收入归星球所有
- 星主可生成邀请码，免费邀请用户加入
```

### 7.3 跟单规则

```
- 星球成员跟单该星球内的信号免费
- 用户需要绑定MT账号才能跟单
- 跟单状态：active（活跃）、paused（暂停）、closed（关闭）
- 每个用户对每个信号源只能有一条跟单记录
```

### 7.4 发布者规则

```
- 星主默认可以发布信号（ownerAsPublisher）
- 其他用户需要申请成为发布者
- 发布者数量受maxPublishers限制
- 发布者申请需要星主审核
```

### 7.5 MT账号绑定规则

```
- 每个用户只能绑定一个MT账号
- 同一个MT账号只能被绑定一次
- MT账号用于EA识别跟单用户
```

### 11.2 常见问题

**Q: 用户可以创建多个星球吗？**
A: 可以，只要星球币余额足够支付创建费用。

**Q: 星球成员可以加入多个星球吗？**
A: 可以，没有限制。

**Q: 跟单需要额外费用吗？**
A: 星球成员跟单该星球内的信号免费。

**Q: MT账号可以更换吗？**
A: 可以，先解绑再重新绑定。

**Q: 信号是如何发送的？**
A: 通过EA调用API发送，平台接收后存储并关联到星球。
